# Claude Code Toolkit — Design Spec

**Дата:** 2026-04-02
**Статус:** Approved
**Подход:** Полная замена текущей системы (4 news-агента → 7 универсальных)

---

## 1. Обзор

Универсальный тулкит из 7 агентов для Claude Code Agent Teams. Каждый агент заточен под конкретный класс задач. Можно вызывать напрямую или через Router, который маршрутизирует запросы и строит цепочки.

Система предназначена для распространения пользователям, у которых может не быть установленных MCP-серверов. Каждый агент проверяет наличие необходимых MCP при старте и предлагает установку с инструкцией. При отказе — работает через fallback.

## 3. Архитектура

### 3.1 Два режима вызова

- **Напрямую:** пользователь создаёт Team с конкретным агентом
- **Через Router:** пользователь описывает задачу произвольно, Router определяет агента(ов) и строит цепочку

### 3.2 Цепочки агентов

Router может строить цепочки, когда задача требует нескольких агентов:

```
"Спарси вакансии и сделай отчёт" → Parser → Report Generator
"Найди новости и проанализируй тренды" → News Digest → Report Generator
"Проанализируй этот PDF и сделай саммари в Sheets" → Doc Analyzer → Report Generator
```

Цепочки не обязательны — каждый агент полностью автономен.

### 3.3 Runtime-директории

```
agent-runtime/
├── shared/        # Промежуточные данные между агентами (для цепочек)
├── outputs/       # Финальные результаты
└── state/         # Статус работы
```

### 3.4 Паттерн MCP-проверки

Единый для всех агентов. При старте:

1. Попробовать вызвать инструмент MCP (например, список tools)
2. Если недоступен → сообщить пользователю:
   ```
   Для [функция] рекомендуется [MCP-сервер].
   Хотите установить? Вот инструкция:
   [пошаговая инструкция установки]
   ```
3. Если пользователь отказывается → использовать fallback
4. Продолжить работу

**Таблица fallback:**

| MCP | Fallback |
|-----|----------|
| Google Sheets | CSV/XLSX через skill `xlsx` |
| Chrome DevTools | WebFetch (с предупреждением о JS-rendered контенте) |

## 4. Детальное описание агентов

### 4.1 Router (lead-агент)

**Назначение:** маршрутизация запросов + построение цепочек
**Модель:** opus
**Инструменты:** SendMessage, TaskCreate, TaskList, TaskUpdate

**Таблица маршрутизации:**

| Триггерные паттерны | Агент |
|---|---|
| "найди", "сравни", "исследуй", "что лучше", "плюсы/минусы" | Deep Research |
| "спарси", "собери с сайта", "таблица с", "заполни sheets", URL-ы | Parser |
| "новости", "дайджест", "что нового", "свежее по теме" | News Digest |
| "проанализируй документ/файл", "прочитай PDF", "суть контракта" | Doc Analyzer |
| "сделай отчёт", "визуализируй", "графики", "PDF из данных" | Report Generator |
| "заметки встречи", "транскрипт", "action items", "митинг" | Meeting Notes |

**Логика:**
1. Классифицирует запрос по таблице маршрутизации
2. Если задача требует нескольких агентов — строит цепочку
3. Запускает агентов последовательно через SendMessage
4. Передаёт промежуточные результаты через `agent-runtime/shared/`
5. Собирает финальный результат и отдаёт пользователю

### 4.2 Deep Research

**Назначение:** глубокий ресёрч по любой теме
**Модель:** opus
**Инструменты:** WebSearch, WebFetch, Read, Write
**MCP:** не требуется (полностью автономный)

**Процесс:**
1. Получает тему/вопрос
2. Формулирует 5-10 поисковых запросов (EN + RU)
3. WebSearch → собирает источники
4. WebFetch → читает полные тексты лучших результатов
5. Синтезирует структурированный отчёт

**Вывод:** `agent-runtime/outputs/research-<topic>.md`

Формат отчёта:
- Краткий ответ / рекомендация
- Таблица сравнения (если применимо)
- Плюсы / минусы
- Детальный анализ
- Источники со ссылками

### 4.3 Parser

**Назначение:** парсинг сайтов в таблицу + автозаполнение Google Sheets
**Модель:** sonnet
**Инструменты:** WebFetch, Read, Write, Bash
**MCP (опционально):** Chrome DevTools, Google Sheets

**Инструкции установки MCP:**

Chrome DevTools:
```
Добавьте в настройки Claude Code MCP-сервер chrome-devtools.
Он позволяет парсить JS-rendered сайты (SPA, динамический контент).
Без него парсинг работает через WebFetch (только статический HTML).
```

Google Sheets:
```
Добавьте в настройки Claude Code MCP-сервер googlesheets.
Он позволяет записывать данные напрямую в Google Sheets.
Без него данные сохраняются в CSV/XLSX файл.
```

**Fallback:**
- Нет Chrome DevTools → WebFetch (предупреждение о JS-rendered контенте)
- Нет Google Sheets → CSV/XLSX через skill `xlsx`

**Процесс:**
1. Получает URL(ы) + описание что извлечь
2. Проверяет доступность MCP
3. Парсит данные (Chrome DevTools или WebFetch)
4. Структурирует в таблицу
5. Сохраняет в Google Sheets или CSV/XLSX

**Вывод:** Google Sheet (ссылка) или `agent-runtime/outputs/parsed-data.xlsx`

### 4.4 News Digest

**Назначение:** дайджест новостей по темам интересов
**Модель:** sonnet
**Инструменты:** WebSearch, WebFetch, Read, Write
**MCP:** не требуется

**Процесс:**
1. Получает темы интересов + период (день/неделя)
2. WebSearch по каждой теме (свежие новости)
3. WebFetch → читает статьи
4. Фильтрует шум, ранжирует по важности
5. Собирает дайджест

**Вывод:** `agent-runtime/outputs/digest-<date>.md`

Формат дайджеста:
- Дата и охваченные темы
- Топ-10 новостей, каждая:
  - Заголовок
  - Источник и дата
  - Саммари (2-3 предложения)
  - Ссылка на оригинал
  - Тег темы

### 4.5 Doc Analyzer

**Назначение:** анализ документов любого формата
**Модель:** opus
**Инструменты:** Read, Write, WebSearch (для контекстного поиска)
**MCP:** не требуется

Поддерживаемые форматы (через нативный Read Claude Code):
- PDF, DOCX, изображения (PNG, JPG), CSV, plain text, код, Jupyter notebooks

**Процесс:**
1. Получает путь к файлу(ам)
2. Читает документ через Read
3. Анализирует содержание
4. При необходимости — WebSearch для контекста (например, проверка юридических терминов)
5. Выдаёт структурированный ответ
6. Готов к интерактивному Q&A по документу

**Вывод:** `agent-runtime/outputs/analysis-<filename>.md`

Формат:
- Тип документа и метаданные
- Краткое саммари (3-5 предложений)
- Ключевые пункты (bullet list)
- Потенциальные проблемы / риски
- Вопросы которые стоит задать
- (для контрактов) важные даты, обязательства сторон

### 4.6 Report Generator

**Назначение:** генерация отчётов из сырых данных
**Модель:** sonnet
**Инструменты:** Read, Write, Bash
**MCP (опционально):** Google Sheets (для графиков через Charts API)
**Skills:** `pdf`, `xlsx`

**Fallback:** Нет Google Sheets → PDF + XLSX, визуализация через HTML-таблицы

**Процесс:**
1. Получает сырые данные (CSV, JSON, текст, заметки, или файл из `agent-runtime/shared/`)
2. Проверяет MCP
3. Анализирует данные, находит паттерны и тренды
4. Генерирует отчёт с визуализацией

**Вывод:** `agent-runtime/outputs/report.pdf` + `report.xlsx` или Google Sheet

### 4.7 Meeting Notes

**Назначение:** обработка транскриптов встреч
**Модель:** sonnet
**Инструменты:** Read, Write
**MCP (опционально):** Google Sheets (для записи action items)

**Fallback:** Нет Sheets → action items в markdown-таблице

**Процесс:**
1. Получает транскрипт (текстовый файл, copy-paste, или документ)
2. Извлекает структуру встречи
3. Генерирует отчёт
4. Опционально записывает action items в Google Sheet

**Вывод:** `agent-runtime/outputs/meeting-<date>.md`

Формат:
- Саммари (3-5 предложений)
- Участники (если упоминаются)
- Принятые решения
- Action items (кто, что, дедлайн) — таблица
- Открытые вопросы
- Ключевые цитаты (если есть важные)

## 7. Примеры использования

### Прямой вызов агента:
```
Создай Team с агентом deep-research.
Тема: "Лучшие ноутбуки для разработки 2026 — сравни MacBook Pro M5 vs ThinkPad X1"
```

### Через Router:
```
Создай Agent Team с router.
Задание: "Спарси топ-30 вакансий Python-разработчика на HH и сделай отчёт с аналитикой по зарплатам"
```
Router определит: Parser → Report Generator

### Цепочка через Router:
```
Создай Agent Team с router.
Задание: "Собери новости по AI за неделю, проанализируй тренды и сделай PDF-отчёт"
```
Router определит: News Digest → Report Generator
